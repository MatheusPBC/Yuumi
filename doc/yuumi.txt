*yuumi.txt* Plan-guided pair programming for Neovim

Yuumi                                                     *yuumi*

Yuumi reads a plan generated outside Neovim and turns it into guided
navigation, buffer marks, hover guidance, a right-side guidance board, and
deterministic inline hints.

The plugin does not generate plans and does not edit files by itself.

QUICK START                                           *yuumi-quick-start*

With lazy.nvim:
>
  {
    "MatheusPBC/Yuumi",
    config = function()
      require("yuumi").setup()
    end,
  }
<

Create or generate a plan at:
>
  .agent/current-plan.json
<

Then run:
>
  :YuumiLoad
  :YuumiBoard
  :YuumiFiles
  :YuumiNext
<

COMMANDS                                                *yuumi-commands*

                                                        *:YuumiLoad*
:YuumiLoad [path]
  Load a Yuumi JSON plan. Defaults to .agent/current-plan.json.

                                                        *:YuumiBoard*
:YuumiBoard
  Show a floating guidance board on the right side of the editor. The board
  lists plan files, anchors, current status, guidance, removal text, and done
  criteria for the active anchor.

                                                        *:YuumiFiles*
:YuumiFiles
  List plan anchors using vim.ui.select.

                                                        *:YuumiNext*
:YuumiNext
  Jump to the next anchor.

                                                        *:YuumiPrev*
:YuumiPrev
  Jump to the previous anchor.

                                                        *:YuumiHover*
:YuumiHover
  Show reason, guidance, and done criteria for the anchor under cursor.

                                                        *:YuumiStatus*
:YuumiStatus
  Show current plan progress in memory.

                                                        *:YuumiDone*
:YuumiDone
  Mark the current anchor as done in memory.

                                                        *:YuumiSkip*
:YuumiSkip
  Mark the current anchor as skipped in memory.

                                                        *:YuumiResetState*
:YuumiResetState
  Clear persisted state from .agent/yuumi-state.json and remove in-memory
  anchor statuses.

                                                        *:YuumiReanchor*
:YuumiReanchor
  Try to move anchors in the current buffer using anchorText, beforeText, or
  afterText from the plan. If no deterministic match is found and a GPT command
  is configured, it falls back to the configured adapter.

                                                        *:YuumiAcceptInline*
:YuumiAcceptInline
  Accept the current deterministic inline suggestion.
  By default the same action is mapped to <M-y> in insert mode.

Floating popups such as :YuumiHover, :YuumiStatus, and :YuumiCheck can be closed
by running the same command again. If the popup is focused, press q.

BLINK.CMP SOURCE                                      *yuumi-blink*

Yuumi exposes a blink.cmp source at module yuumi.blink. Add it as a provider:
>
  sources = {
    default = { "lsp", "path", "snippets", "buffer", "yuumi" },
    providers = {
      yuumi = { name = "Yuumi", module = "yuumi.blink" },
    },
  }
<

The source returns inlineSuggestions from the loaded plan for the current file.

GPT COMMANDS                                        *yuumi-gpt-commands*

These commands use a configured command adapter when gpt_command is set.
The command receives a JSON payload on stdin and should write text to stdout.
Yuumi does not store API keys. Keep provider credentials in environment
variables such as OPENAI_API_KEY or delegate to OpenCode/local wrappers.
>
  :YuumiExplain
  :YuumiSuggest
  :YuumiCheck
  :YuumiBreakdown
<

PLAN CONTRACT                                      *yuumi-plan-contract*

Minimum plan shape:
>
  {
    "version": 1,
    "title": "Example plan",
    "tasks": [
      {
        "id": "task-1",
        "file": "src/example.lua",
        "summary": "Manual edit guidance",
        "anchors": [
          {
            "id": "anchor-1",
            "line": 10,
            "endLine": 12,
            "anchorText": "exact text to find if line numbers drift",
            "beforeText": "nearby text before the anchor",
            "afterText": "nearby text after the anchor",
            "reason": "Why this region matters",
            "guidance": "What the developer should do manually",
            "writeText": [
              "local value = compute_value(input)",
              "return value"
            ],
            "doneWhen": ["Expected behavior is implemented"],
            "inlineSuggestions": [
              {
                "trigger": "local",
                "insertText": "local value = compute_value(input)"
              }
            ]
          }
        ]
      }
    ]
  }
<

CONFIGURATION                                      *yuumi-configuration*

Default setup:
>
  require("yuumi").setup({
    plan_path = ".agent/current-plan.json",
    state_path = ".agent/yuumi-state.json",
    highlight_group = "YuumiAnchor",
    virtual_text_prefix = "yuumi: ",
    virtual_text_pos = "right_align",
    show_virtual_lines = true,
    open_files_on_load = true,
    inline_debounce_ms = 80,
    inline_ai_enabled = false,
    accept_keymap = "<M-y>",
    gpt_command = nil,
  })
<

When open_files_on_load is enabled, :YuumiLoad opens the board before the file
picker. The default virtual_text_pos is right_align so guidance does not look
like typed code.

INLINE GUIDANCE                                      *yuumi-inline-guidance*

Yuumi inline guidance tries writeText first, inlineSuggestions second, and AI
fallback last. writeText can complete the rest of the current line or the next
missing line from the planned block without an exact trigger.

AI fallback is off by default. Enable it with inline_ai_enabled = true and set
gpt_command. Yuumi sends an InlineSuggest JSON payload with file, cursor line,
current prefix, nearby lines, guidance, and writeText. The command should return
only the text suffix to insert at the cursor.
